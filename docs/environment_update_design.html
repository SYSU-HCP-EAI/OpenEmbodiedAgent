ENVIRONMENT.md 写入更新设计图

基于当前实现：感知结果写入 agent/shared workspace 的 ENVIRONMENT.md；普通 runtime session 摘要写入 runtime workspace 的 LOG.md，不污染环境状态。

Workspace 写入边界

runtime workspace

输入协议

TARGETS.md、SKILLS.md、SESSIONS.md、sensor/perception YAML。

WatchdogSupervisor

执行 session 调度、preflight、target reset、policy/skill runtime。

LOG.md

保存 runtime session history：last session、status、artifact_dir、num_steps 等。

→
agent/shared workspace

ENVIRONMENT.md

仅由 perception v2 writer 写入环境对象、关系、map、tf、perception runs。

Agent 读取入口

上层 Agent、scene graph tool、embodied critic 应读取这个 shared workspace。

启动参数

--environment-workspace 指向 shared workspace；不传则默认等于 runtime workspace。

ENVIRONMENT.md 更新流水线

1

读取现有文件

如果不存在，创建空 v2 文档；如果不是 PhyAgentOS.environment.v2，旧内容不迁移。

2

解析 delta

插件输出 EnvironmentDelta.objects，key 是插件本地 object id。

3

解析 scope

优先使用 delta.scope，否则使用 delta.refresh_scope 或本次 object ids。

4

全局 id 匹配

同 label、同 frame、3D 距离小于阈值时复用已有 global id。

5

对象合并

匹配则 update；不匹配则生成 target_id::local_id 并 insert。

6

关系重写

关系中的 local id 会尽量改写为 global id；引用缺失对象的旧关系会被移除。

7

原子写入

写 JSON fenced block 到 ENVIRONMENT.md，使用临时文件 + replace。

对象更新决策

Update：重复物体
  • 已有对象和新对象 label 相同。
  • pose.frame_id 相同。
  • position_m 距离 ≤ position_threshold_m，默认 0.25m。
  • 复用已有 global id，更新对象内容与 source。
Insert：新物体
  • 没有可用 3D anchor，或没有命中已有对象。
  • 生成 <target_id>::<local_object_id>。
  • 若命名冲突，追加数字后缀。
Delete：scope 内确认缺失
  • 只考虑同一 target_id 来源的对象。
  • 对象在 resolved refresh scope 内。
  • 本次 delta 没有再输出该对象。
  • 删除对象后，引用它的关系也会被清理。

失败路径与不会写入的内容

Preflight 失败
  • session 标记为 rejected。
  • 不进入 running。
  • 不执行 policy 或 skill runtime。
  • 不写入 perception objects。
普通 runtime summary
  • 不再写入 ENVIRONMENT.md。
  • 改写到 runtime workspace 的 LOG.md。
  • 用于临时 session history。
大文件与 artifact
  • 不把 raw image、mask、depth、pointcloud、tensor 写进 Markdown。
  • 大文件应写到 artifacts。
  • 环境文件只保存 compact summary 和 URI。

当前设计建议

多 target 共享一个 ENVIRONMENT.md 时，插件应尽量输出 metric pose。没有 pose.frame_id 和 pose.position_m 的对象无法通过位置阈值去重，只能按 target_id::local_id 插入。真实机器人接入时应统一 TF frame，例如 map 或 base_link，否则跨 target 物体不会被判定为同一对象。
