PhyAgentOS 感知模块结构图

基于当前代码实现：runtime workspace 负责协议与执行，agent/shared workspace 接收感知写入的 ENVIRONMENT.md，普通 session 摘要写入 LOG.md。

模块总览

TARGETS.md、SKILLS.md、SESSIONS.md、LOG.md

configs/runtime/perception/<target>.perception.yaml
声明 models、plugin_candidates、pipelines。

runtime/perception

PerceptionRuntime

runtime/perception/perception_runtime.py
协调 resolver、preflight、frame builder、pipeline、writer。

ConfigResolver + Preflight

解析 YAML；严格检查 sensor、pipeline、plugin、model、checkpoint、endpoint。

SensorFrameBuilder

从 target observation 中按 observation_key 取通道，并校验 dtype/shape。

PluginPipeline

按 pipeline 的 use_plugins 顺序执行插件，生成 EnvironmentDelta。

EnvironmentWriter

负责全局 object id、refresh scope、关系重写、原子写入 ENVIRONMENT.md。

插件与状态输出

perception_plugins

BasePerceptionPlugin、dummy_segmenter、rgbd_object_builder、sim_oracle。

重模型懒加载插件

SAM、YOLO、Open3D、ROS、LiDAR SDK 只能在 preflight() 或 run() 中懒加载。

agent/shared workspace

ENVIRONMENT.md 写入这里。上层 Agent 应读取同一个 shared workspace。

runtime workspace

LOG.md 保存 session history，artifacts/runtime 保存 episode。

协议/配置文件 runtime/perception 代码 感知插件 状态输出 严格拒绝路径

Session 进入感知的执行链路

1

Watchdog claim

WatchdogSupervisor.run_once() 选中 pending session 并 claim。

2

Health preflight

检查 target enabled、timeout、execution、policy endpoint 等轻量 runtime 条件。

3

Resolve perception plan

根据 skill requirements 决定是否需要感知，并解析 sensor/perception YAML。

4

Strict preflight

缺 sensor、model、pipeline、checkpoint、endpoint 时直接 reject，不进入 running。

5

Build frame + plugins

从 target observation 构造 SensorFrame，按 pipeline 顺序执行插件。

6

Write state

感知结果写入 shared ENVIRONMENT.md；session 结果写入 runtime LOG.md。

关键文件职责

文件
职责
用户需要关注什么
runtime/watchdog/supervisor.py
串起 session 调度、health preflight、perception preflight、target/policy/skill execution。
启动时可传 --environment-workspace，确保 Agent 能读到环境文件。
runtime/perception/config_resolver.py
读取 target/skill，并加载 sensor/perception YAML，选择满足 outputs 的 pipeline。
确认 TARGETS.md 中 config ref 路径正确。
runtime/perception/preflight.py
执行严格准入检查，失败时构造可操作错误。
根据 reject 信息修 sensor、模型、pipeline 或 endpoint。
runtime/perception/sensor_frame_builder.py
把 target observation 映射为插件可读的 SensorFrame。
observation_key 必须和 target observation 实际字段匹配。
runtime/perception/plugin_pipeline.py
实例化并按顺序运行 perception plugins。
插件 id 必须是内置插件，或提供 importable module/class。
runtime/perception/environment_writer.py
合并对象、分配全局 id、处理 refresh scope、重写关系并写 ENVIRONMENT.md。
插件应输出稳定 local id 和可用于锚定的 pose.frame_id/position_m。
plugins/perception_plugins/base.py
插件基类，定义 preflight() 和 run()。
重依赖必须懒加载，插件输出 EnvironmentDelta。

当前实现边界

当前版本是 dependency-free MVP：可以验证协议、preflight、插件流水线和 ENVIRONMENT.md 写入。真实模型服务、3D 几何重建、跨 target 高级 identity fusion 需要通过插件继续扩展。
