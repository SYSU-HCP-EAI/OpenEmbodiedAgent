BuiltinSkillRuntime 执行通路

从上层提交 builtin session，到 watchdog 完成调度和 preflight，再由 SessionRunner 将受控 target handle 交给 builtin runtime 执行内置逻辑或 agent-interactive target tools。

配置与指令 watchdog 调度 session/runtime 边界 builtin skill 逻辑 target 执行链路 结果落盘

端到端总览

上层输入

SESSIONS.md

用户任务进入 pending session：包含 task_description、target_ref、skill_ref、执行步数、超时和 runtime hints。

SKILLS.md

通过 runtime_kind: builtin 选择 builtin 路径；可设置 agent_exposure 与 target_tool_policy 暴露受控 target tools。

TARGETS.md

声明 target runtime、target adapter、supported skills、observation 语义和可执行动作契约。

→
调度与检查

WatchdogSupervisor.run_once()

读取 runtime 文档，调用 scheduler 选择 pending session，并通过 registry 抢占 session。

SessionScheduler.resolve_session()

解析 target 与 skill 引用，检查支持关系。builtin skill 可以使用 empty observation，但必须双方显式允许。

RuntimeCompatibilityPreflight.check()

校验 target、runtime registry、adapter stack、action bridge、sensor；如果暴露 target tools，会产出 TargetToolManifest。

→
运行时装配

构建 target

TargetRuntimeRegistry.build() 根据 target class 构建本地 target 或 remote target proxy。

构建 builtin runtime

SkillRuntimeRegistry.build() 按 skill 的 runtime 名称创建具体 BuiltinSkillRuntime 实现。

无 policy client

builtin 路径不会调用 build_policy_client()，动作选择由 builtin runtime 自己的逻辑、脚本或 agent loop 决定。

→
SessionRunner

SessionRunner.start()

构建 adapter stack，执行 target.build()、configure_session()、start_session()、reset()。

TargetSessionHandle

将 target 原始对象包成受控接口，并携带 adapter plan、session state、perception runtime、target tool manifest。

进入 builtin 分支

当 runtime_kind != "policy" 时，检查 runtime 类型并调用 run_builtin_loop()。

→
执行与结果

BuiltinSkillRuntime.run_builtin_loop()

具体 runtime 根据 task、runtime hints、环境状态或 agent 输出决定观察、动作、tool call 和终止条件。

受控 target 操作

通过 observe()、action_chunk()、call_target_tool()、request_environment_refresh() 操作底层 target。

结果落盘

builtin runtime 返回 SkillRuntimeResult，watchdog 写 episode、session history，并更新 session 状态。

Builtin 执行内部步骤

1

skill_runtime.start()

初始化一次 session 的 skill-local 状态；SessionRunner 同时维护 heartbeat、trace id 和 step index。

2

run_builtin_loop()

builtin runtime 接收 SkillContext、TargetSessionHandle 和 AdapterPlan，不接收 policy client。

3

观察或空观察

target_handle.observe() 可返回 target adapter 处理后的 runtime observation；若双方显式允许 empty observation，则返回空观察。

4

内置决策

runtime 可使用固定规则、动作计划、环境状态或 agent-interactive loop 生成 skill output/action chunk。

5

动作或工具调用

动作路径经 action bridges 与 target adapter 后调用 target.action_chunk()；工具路径先受 TargetToolManifest 白名单约束。

6

状态更新

handle 更新 last_status、step_index 和 heartbeat；runtime 可通过 execution_status() 获取底层执行状态。

7

返回结果

达到 success、done、失败、取消或 max steps 后返回 SkillRuntimeResult，由 SessionRunner 转为 session result。

关键边界

Target tools 暴露受 preflight 约束

当 builtin skill 设置 agent_exposure != "none" 时，必须配置 target_tool_policy。preflight 会过滤 reset、close、raw_sdk_command、disable_safety 等危险工具。

Handle 是唯一 target 边界

builtin runtime 通过 handle 访问 target；handle 负责 adapter 转换、action bridge、perception refresh、tool manifest 校验和 session heartbeat。

当前代码状态

仓库中 BuiltinSkillRuntime 是抽象基类，具体 builtin runtime 需要继承它并注册到 SkillRuntimeRegistry。
